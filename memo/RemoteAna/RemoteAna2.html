リモート解析環境/Webブラウザのプロキシ設定

Memo-リモート解析環境へ戻る

自動構成スクリプトを書いてWebブラウザの設定をすると、一々手動でプロキシ設定を切り替えなくても自動で切り替えてくれるようにできる。

自動構成スクリプト

以下の例のようなスクリプトを書いて.pac ファイルとして保存し、これを後述の方法でブラウザから指定する。 これにより、自分の接続しているネットワークや見たいサイトのドメインなどの条件によってプロキシの設定を自動で変更するようにできる。


function FindProxyForURL(url, host)
{
    // 研究室にいるときの設定
    //　自分のIPが192.168.**.**だったら全てダイレクトで接続
    if(myIpAddress().substr(0,8) == "192.168."){
        return "DIRECT";
    }

    // その他にいるときの設定
    // 研究室のローカルネットワーク内のホストへはゲートウェイサーバーを経由
    if(dnsDomainIs(host,"[server name1]"))
            return "SOCKS localhost:1080";
    if(dnsDomainIs(host,"[server name2]"))
            return "SOCKS localhost:1080";
    //論文系のサイトも研究室のサーバー経由
    if(dnsDomainIs(host,".aip.org"))
            return "SOCKS localhost:1080";
    if(dnsDomainIs(host,".aps.org"))
            return "SOCKS localhost:1080";
    if(dnsDomainIs(host,".iop.org"))
            return "SOCKS localhost:1080";
    if(dnsDomainIs(host,".ieee.org"))
            return "SOCKS localhost:1080";
    if(dnsDomainIs(host,".sciencedirect.com"))
            return "SOCKS localhost:1080";
    if(dnsDomainIs(host,".springerlink.com"))
            return "SOCKS localhost:1080";
    //それ以外はダイレクト
        return "DIRECT";
}

Webブラウザの設定

IE9の場合

[右上の歯車ボタン]→[インターネットオプション]→[接続]タブ→[LANの設定]→[自動構成スクリプトを使用する]にチェック

FireFoxの場合

[ツール]あるいは[左上のFireFox]→[オプション]→[詳細]タブ→[ネットワーク]タブ→[接続設定]→[自動プロキシ設定スクリプト]にチェック

Chromeの場合

CHrome はコンピューターのプロキシ設定を使っているみたいなので、インターネットオプションを変更すればよい。

Safariの場合

[右上の歯車ボタン]→[設定]→[詳細]タブ→プロキシ：[設定を変更]

いずれの場合もアドレス入力欄に、file://C:/[file path].pac の形で自動構成スクリプトを指定する。

Windows の場合、[インターネットオプション]→[セキュリティ]タブ→[ローカルイントラネット]→[サイト]→[プロキシサーバーを利用しないサイトを全て含める]のチェックを外す。
そうしないと、return "DIRECT"; したサイトが全てローカルネットワークと同じセキュリティ設定として扱われてしまうらしい。 これは、そもそも「ローカルネットワークにしかつながっていないPCから外部のサイトを見るときにゲートウェイサーバーを経由する」という状況で使われることを想定しているからのようです。
